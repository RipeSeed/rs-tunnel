# Security Policy

## Reporting a Vulnerability

Please do not disclose security vulnerabilities in public issues.

Use GitHub Security Advisories for private reporting:

- <https://github.com/RipeSeed/rs-tunnel/security/advisories/new>

If advisory access is unavailable, contact the maintainers directly and include:

- A clear description of the issue
- Reproduction steps or proof of concept
- Potential impact
- Suggested remediation (if known)

## Response Expectations

- Initial acknowledgment target: within 3 business days
- Triage/update target: within 7 business days
- Fix timelines depend on severity and release risk

## Scope

In-scope concerns include (not exhaustive):

- Authentication or authorization bypass
- Token/credential leakage
- Sensitive data exposure in logs or responses
- SSRF, command injection, or privilege escalation
